# 部署指南 — 怎麼把工具發給同學

目標：讓同學**用一個網址就能開**，不用安裝任何東西。
這個 App 是純前端靜態網站，可以放在免費靜態託管上。以下三種方式，推薦 **方式 A**。

---

## 方式 A：GitHub Pages（免費、最簡單）✅ 推薦

**前提**：repo 需設為 **公開（public）**。公開的只有「程式碼」，**不含任何病歷資料**（病歷只在使用者自己瀏覽器處理），所以公開是安全的。

1. 在 GitHub 建一個新 repo（例如 `vet-record-generator`）。
2. 把本專案推上去：
   ```bash
   cd D:\claude\github\vet-record-generator
   git remote add origin https://github.com/<你的帳號>/vet-record-generator.git
   git branch -M main
   git push -u origin main
   ```
3. 到 repo 的 **Settings → Pages**：
   - Source 選 **Deploy from a branch**
   - Branch 選 **main**、資料夾選 **/(root)**，按 Save。
4. 等 1–2 分鐘，頁面會給你一個網址：
   `https://<你的帳號>.github.io/vet-record-generator/`
5. 把這個網址貼給同學就完成了。

**之後要更新**：改完 code 後 `git push`，GitHub Pages 會自動重新部署。

---

## 方式 B：Cloudflare Pages（可用私有 repo）

若你不想把 repo 公開：
1. repo 先推上 GitHub（可設私有）。
2. 到 **dash.cloudflare.com → Workers & Pages → Create → Pages → Connect to Git**。
3. 選這個 repo；build 設定留空（沒有 build 步驟），輸出目錄填 `/`。
4. 部署完會給一個 `*.pages.dev` 網址，貼給同學。

---

## 方式 C：本機執行（不建議給非技術同學）

沒有網路託管時的臨時方案，但同學要自己裝 Python + 開終端機，門檻高：
```bash
cd 專案資料夾
python -m http.server 8000
```
再開 `http://localhost:8000`。只適合你自己或技術同學測試用。

---

## 重要：關於「BYO API key」

- 這個 App **不含**任何 API key，也**不需要**你付 AI 費用。
- **每位同學各自申請、各自貼上自己的金鑰**（見《使用手冊》），費用算在各自帳號。
- 所以你發佈網址時，不會外洩任何金鑰，也不會替全班付費。
