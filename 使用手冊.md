# 病歷自動生成器 — 使用手冊

把手寫病歷照片，用 AI 自動轉成醫院格式的 Word 病歷，省下手打時間。
全程在你自己的瀏覽器完成，病歷資料不會上傳到任何我們的伺服器。

---

## 一、開始前：準備一把 AI 金鑰（API key）

這個工具的 AI 辨識/語音功能，需要一把「AI 金鑰」。金鑰只會存在你自己的電腦，別人看不到。
兩種選擇，擇一即可：

### 選項 A：Google Gemini（有免費額度，先試這個）
1. 開瀏覽器到 **aistudio.google.com**，用 Google 帳號登入。
2. 點 **「Get API key」→「Create API key」**。
3. 複製那串以 `AIza...` 開頭的金鑰，先貼在記事本備用。

> ⚠️ 若之後辨識時出現「額度上限 / limit: 0」的錯誤，代表你的 Google 帳號拿不到免費額度，請改用選項 B。

### 選項 B：OpenAI（穩定，付費，一份病歷約幾美分）
1. 到 **platform.openai.com** 登入。
2. 左側 **Settings → Billing**，儲值約 **US$5**（夠用很久）。
3. 左側 **API keys → Create new secret key**，複製 `sk-...` 開頭的金鑰。

## 二、開啟工具

- **如果班上已經有網址**：直接用瀏覽器（建議 Chrome / Edge）打開那個網址即可。
- 手機也能用，但拍照上傳、拉框比較適合在電腦上操作。

---

## 三、第一次設定金鑰（只需做一次）

1. 點右上角 **「⚙ 設定」**。
2. **AI 供應商**：選你申請的那家（Google Gemini 或 OpenAI）。
3. 把金鑰貼進 **API key** 欄。
4. 模型維持預設即可。
5. 按 **儲存**。

金鑰會記在這台瀏覽器，下次不用再貼（除非換電腦或清除瀏覽器資料）。

## 四、產一份病歷（4 步驟）

### 步驟 1：選格式
依這份病例的主治老師選對應格式：
- **預設外科格式**
- **外科-劉乃潔老師**（版面/欄位和預設不同）

選好按「下一步」。

### 步驟 2：上傳照片並「去識別化」
> 🔒 **很重要**：真實病歷是敏感資料。送 AI 前，一定要先把飼主個資蓋掉。

1. 按選檔，選手寫病歷照片。**一份病歷有兩張時可以一次或分次都選進來**。
2. 在每張照片上，用滑鼠**按住拖曳**畫黑框，蓋掉：
   - 飼主姓名、電話、地址
   - 病歷號、晶片號
3. 框錯了可按「復原上一格」或「清除全部黑框」。
4. 蓋好後按 **「🔎 用 AI 辨識照片並填入欄位」**，等幾秒，欄位會自動填好。
   - 沒有照片、想純手打的話，按「略過」即可。

### 步驟 3：檢查與補充欄位
- AI 填好的內容會出現在各欄位，**逐欄檢查、直接修改**。
- 內容會自動翻成英文；醫院名、寵物名、飼料品牌會保留中文。
- **想用講的補充**：點欄位旁的 **🎤**，說話（可以講中文），再按一次停止，內容會自動轉成英文填進去。
  - 第一次會問「允許使用麥克風」，按允許。

填好按「下一步」。

### 步驟 4：確認並匯出
1. 檢視整份內容，若有紅字提醒缺必填（CC / HP / PE / A/P），建議補齊。
2. 按 **「匯出 .docx」**，瀏覽器會下載一份 Word 檔。
3. 用 Word 打開，最後再校對一次，就能上交。

> 匯出需要連網（載入產生 Word 的元件）。

## 五、常見問題

**Q：辨識時出現「額度上限 / limit: 0 / 429」？**
Gemini 免費額度不足或該帳號沒有免費額度。改到「⚙ 設定」換 OpenAI，或改天再試。

**Q：出現「API key 無效 / 401 / 400」？**
金鑰貼錯或過期。到「⚙ 設定」重新貼一次；確認供應商選對（Gemini 的 key 不能給 OpenAI 用，反之亦然）。

**Q：辨識結果不準？**
手寫太潦草時 AI 會猜錯，這是正常的——**最後一定要人工校對**。照片越清晰、字越工整，辨識越準。可用 🎤 口述補正。

**Q：語音轉錄失敗？**
先確認已允許麥克風。若用 Gemini 仍失敗，可改用 OpenAI。

**Q：金鑰不見了？**
換電腦、換瀏覽器、或清除瀏覽器資料後要重新貼一次金鑰。

## 六、隱私與責任提醒

- 送 AI 前**務必**把飼主個資塗黑（步驟 2）。
- AI 產出僅供輔助，**最終內容由你本人負責校對**，不要照單全收。
- 你的金鑰、病歷內容只在你的瀏覽器與你選的 AI 供應商之間，不經過我們的伺服器。
- 使用免費 Gemini 時，Google 可能將送出的內容用於改善產品（即使已去識別化）；很在意的話請用 OpenAI。
